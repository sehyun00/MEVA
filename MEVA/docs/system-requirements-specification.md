## 1. 개요

### 1.1 목적
본 문서는 재료공학 시각화 및 분석 시스템(MEVA)의 시스템 요구사항을 정의한다. 본 명세서는 시스템 설계, 구현, 테스트의 기준이 되며, 프로젝트 이해관계자들 간의 의사소통 도구로 활용된다.

### 1.2 범위
MEVA는 재료공학 실험 데이터를 시각화하고 분석하는 Java 기반 데스크톱 애플리케이션이다. 주요 기능은 다음과 같다:
- 인장 시험 데이터의 응력-변형률 곡선 시뮬레이션
- 재료 물성값 자동 계산 및 분석
- 실험 데이터 관리 및 시각화
- 분석 결과 보고서 생성 및 내보내기

### 1.3 대상 사용자
- 재료공학과 학부생 및 대학원생
- 재료 실험 연구원
- 재료공학 교육자

### 1.4 참조 문서
- MEVA 프로젝트 요구사항 정의서 v1.0
- 시스템 아키텍처 설계 문서
- 데이터베이스 ERD 설계 문서
- 데이터 플로우 다이어그램

## 2. 시스템 개요

### 2.1 시스템 설명
MEVA는 재료공학 분야에서 인장 시험 데이터를 처리하고 분석하는 통합 시스템이다. 사용자가 재료의 물성값을 입력하면 시스템이 이론적 응력-변형률 곡선을 생성하고, 주요 물성값을 자동으로 계산하여 시각화한다.

### 2.2 시스템 특징
- **직관적 GUI**: Java Swing 기반의 사용자 친화적 인터페이스
- **실시간 시각화**: JFreeChart를 활용한 동적 그래프 생성
- **정확한 계산**: 재료공학 이론에 기반한 물성값 계산 알고리즘
- **데이터 관리**: SQLite 기반의 경량 데이터베이스
- **보고서 생성**: PDF 형식의 분석 결과 보고서 출력

### 2.3 주요 컴포넌트
1. **GUI 모듈**: 사용자 인터페이스 및 입력 처리
2. **계산 엔진**: 물성값 계산 및 시뮬레이션 로직
3. **시각화 모듈**: 그래프 및 차트 생성
4. **데이터 관리 모듈**: 데이터베이스 CRUD 연산
5. **파일 처리 모듈**: 데이터 입출력 및 보고서 생성
6. **검증 모듈**: 입력 데이터 유효성 검증

## 3. 기능 요구사항

### 3.1 시뮬레이션 기능 (SIM)

#### FR-SIM-001: 응력-변형률 곡선 시뮬레이션
- **설명**: 사용자 입력값(영률, 항복강도, 인장강도, 시편 치수)을 기반으로 응력-변형률 곡선을 계산한다.
- **우선순위**: 높음
- **입력**: 
  - 영률 (Young's Modulus, GPa)
  - 항복강도 (Yield Strength, MPa)
  - 인장강도 (Ultimate Tensile Strength, MPa)
  - 시편 치수 (직경 또는 단면적, mm)
- **출력**: 응력-변형률 곡선 데이터 포인트
- **처리 규칙**:
  - 탄성 영역: 후크의 법칙 적용 (σ = E × ε)
  - 소성 영역: 경화 모델 적용
  - 파단점까지의 곡선 생성
- **검증 방법**: 다양한 재료 물성값 입력 후 이론적 곡선 생성 확인
- **담당 브랜치**: feature/stress-strain-calculation

#### FR-SIM-002: 단계별 재료 거동 시뮬레이션
- **설명**: 재료의 탄성, 항복, 소성경화, 파단 과정을 단계별로 시뮬레이션한다.
- **우선순위**: 중간
- **단계**:
  1. 탄성 변형 구간 (0 ~ 항복점)
  2. 항복 현상 (항복점)
  3. 소성 경화 구간 (항복점 ~ 인장강도)
  4. 네킹 및 파단 구간 (인장강도 ~ 파단점)
- **출력**: 각 단계별 특성값 및 곡선
- **검증 방법**: 각 단계별 물성값 변화 확인 및 이론값과 비교
- **담당 브랜치**: feature/stress-strain-calculation
- **관련 요구사항**: FR-SIM-001, FR-VIS-002

### 3.2 시각화 기능 (VIS)

#### FR-VIS-001: 응력-변형률 곡선 시각화
- **설명**: JFreeChart를 사용하여 응력-변형률 곡선을 실시간으로 그래프 시각화한다.
- **우선순위**: 높음
- **기능**:
  - X축: 변형률 (Strain, %)
  - Y축: 응력 (Stress, MPa)
  - 실시간 그래프 업데이트
  - 확대/축소 및 팬 기능
- **성능 기준**: 입력값 변경 후 0.5초 이내 그래프 업데이트
- **검증 방법**: 입력값 변경 시 그래프 즉시 업데이트 확인
- **담당 브랜치**: feature/graph-visualization
- **관련 요구사항**: FR-SIM-001, UI-GUI-001

#### FR-VIS-002: 물성값 마커 표시
- **설명**: 그래프 상에 주요 물성값(영률, 항복강도, 인장강도)을 마커로 표시한다.
- **우선순위**: 중간
- **표시 항목**:
  - 비례한도점
  - 항복점
  - 최대 응력점 (인장강도)
  - 파단점
- **표시 방식**: 각 점에 라벨과 좌표값 표시
- **검증 방법**: 계산된 물성값이 그래프에 정확히 표시되는지 확인
- **담당 브랜치**: feature/graph-visualization
- **관련 요구사항**: FR-CALC-001, FR-VIS-001

### 3.3 계산 기능 (CALC)

#### FR-CALC-001: 주요 물성값 자동 계산
- **설명**: 입력된 실험값으로부터 영률, 항복강도, 인장강도 등 주요 물성값을 자동 계산한다.
- **우선순위**: 높음
- **계산 항목**:
  - 영률 (Young's Modulus, E)
  - 0.2% 오프셋 항복강도
  - 인장강도 (UTS)
  - 파단 응력
- **계산 공식**:
  - 영률: E = Δσ / Δε (탄성 구간)
  - 항복강도: 0.2% 오프셋 방법
  - 응력: σ = F / A₀
  - 변형률: ε = (L - L₀) / L₀
- **정밀도**: 소수점 이하 3자리
- **검증 방법**: 표준 재료의 알려진 물성값과 계산 결과 비교
- **담당 브랜치**: feature/stress-strain-calculation
- **관련 요구사항**: FR-SIM-001, DB-DATA-001

#### FR-CALC-002: 추가 물성값 계산
- **설명**: 연신율, 단면수축률 등 추가 물성값을 계산한다.
- **우선순위**: 중간
- **계산 항목**:
  - 연신율 (Elongation, %): (Lf - L₀) / L₀ × 100
  - 단면수축률 (Reduction of Area, %): (A₀ - Af) / A₀ × 100
  - 탄성 계수
  - 인성 (Toughness, 곡선 하부 면적)
- **검증 방법**: 계산 공식의 정확성 및 결과값 검증
- **담당 브랜치**: feature/stress-strain-calculation
- **관련 요구사항**: FR-CALC-001, FR-EXPORT-001

### 3.4 파일 처리 기능 (FILE)

#### FR-FILE-001: 실험 데이터 파일 가져오기
- **설명**: CSV, Excel 형식의 실험 데이터 파일을 읽어들인다.
- **우선순위**: 중간
- **지원 형식**: CSV (.csv), Excel (.xlsx, .xls)
- **데이터 형식**:
  - 필수 열: 하중(N), 변위(mm)
  - 선택 열: 시간(s), 온도(°C)
- **오류 처리**:
  - 파일 형식 불일치
  - 데이터 누락
  - 잘못된 데이터 형식
- **검증 방법**: 다양한 형식의 파일 가져오기 성공 여부 확인
- **담당 브랜치**: feature/file-handler

#### FR-FILE-002: 데이터 내보내기
- **설명**: 분석 결과를 CSV, Excel 형식으로 내보낸다.
- **우선순위**: 낮음
- **내보내기 항목**:
  - 응력-변형률 데이터
  - 계산된 물성값
  - 그래프 이미지 (PNG)
- **검증 방법**: 내보낸 파일의 데이터 정확성 확인
- **담당 브랜치**: feature/file-handler

### 3.5 검증 기능 (VALID)

#### FR-VALID-001: 입력 데이터 유효성 검증
- **설명**: 사용자 입력값의 유효성을 실시간으로 검증한다.
- **우선순위**: 높음
- **검증 항목**:
  - 수치 범위 (양수, 최대/최소값)
  - 데이터 형식 (숫자, 텍스트)
  - 필수 입력 필드
  - 물리적 타당성 (예: 항복강도 < 인장강도)
- **오류 메시지**: 각 검증 실패 시 구체적인 오류 메시지 표시
- **검증 방법**: 잘못된 입력값에 대한 오류 메시지 표시 확인
- **담당 브랜치**: feature/data-validation

#### FR-VALID-002: 계산 결과 검증
- **설명**: 계산된 물성값의 물리적 타당성을 검증한다.
- **우선순위**: 중간
- **검증 규칙**:
  - 물성값 범위 검증 (재료별 일반적 범위)
  - 공학적 상식 검증
  - 이상치 탐지
- **경고 시스템**: 의심스러운 결과에 대한 경고 표시
- **검증 방법**: 비정상 데이터 입력 시 경고 메시지 확인
- **담당 브랜치**: feature/data-validation

### 3.6 내보내기 기능 (EXPORT)

#### FR-EXPORT-001: 결과 데이터 내보내기
- **설명**: 분석 결과를 다양한 형식으로 내보낸다.
- **우선순위**: 중간
- **지원 형식**:
  - CSV: 수치 데이터
  - Excel: 서식 있는 표
  - 이미지: PNG, JPEG (그래프)
- **내보내기 옵션**:
  - 전체 데이터 / 선택 데이터
  - 그래프 포함 여부
- **담당 브랜치**: feature/export-functionality

#### FR-EXPORT-002: PDF 보고서 생성
- **설명**: 시뮬레이션 결과와 그래프를 포함한 PDF 보고서를 생성한다.
- **우선순위**: 낮음
- **보고서 구성**:
  1. 표지: 프로젝트 정보
  2. 입력 데이터 요약
  3. 응력-변형률 그래프
  4. 계산된 물성값 표
  5. 결론 및 분석
- **템플릿**: 사용자 정의 가능한 보고서 템플릿
- **검증 방법**: 생성된 PDF의 내용 완전성 및 레이아웃 확인
- **담당 브랜치**: feature/export-functionality
- **관련 요구사항**: FR-EXPORT-001, FR-CALC-002

## 4. 비기능 요구사항

### 4.1 성능 요구사항

#### NFR-PERF-001: 응답 시간
- **설명**: 시스템의 주요 기능이 신속하게 응답해야 한다.
- **성능 기준**:
  - 데이터 입력 후 그래프 생성: 1초 이내
  - 계산 수행: 0.5초 이내
  - 파일 가져오기 (1MB): 2초 이내
  - GUI 반응성: 0.1초 이내
- **측정 방법**: 성능 프로파일링 도구 사용
- **관련 요구사항**: FR-VIS-001, DB-DATA-001

#### NFR-PERF-002: 확장성
- **설명**: 시스템은 대량의 데이터를 효율적으로 처리할 수 있어야 한다.
- **성능 기준**:
  - 최대 데이터 포인트: 100,000개
  - 메모리 사용량: 최대 512MB
  - 동시 그래프 표시: 최대 5개

### 4.2 사용성 요구사항

#### NFR-USE-001: 학습 용이성
- **설명**: 재료공학 기초 지식이 있는 사용자가 30분 이내에 시스템을 사용할 수 있어야 한다.
- **지원 기능**:
  - 툴팁 및 도움말
  - 샘플 데이터 제공
  - 사용 가이드
- **측정 방법**: 사용자 수용 테스트를 통한 학습 시간 측정

#### NFR-USE-002: 접근성
- **설명**: 다양한 사용자가 시스템을 쉽게 사용할 수 있어야 한다.
- **요구사항**:
  - 명확한 레이블 및 버튼 텍스트
  - 키보드 단축키 지원
  - 적절한 폰트 크기 (최소 12pt)
  - 색맹 친화적 색상 팔레트

### 4.3 신뢰성 요구사항

#### NFR-REL-001: 정확성
- **설명**: 계산 결과의 정확성을 보장해야 한다.
- **정확도 기준**:
  - 물성값 계산: ±1% 이내
  - 그래프 표시: 픽셀 단위 정밀도
- **검증 방법**: 표준 재료 데이터와의 비교 검증

#### NFR-REL-002: 오류 복구
- **설명**: 시스템 오류 발생 시 데이터 손실 없이 복구할 수 있어야 한다.
- **복구 메커니즘**:
  - 자동 저장 (5분 간격)
  - 세션 복원 기능
  - 오류 로그 기록

### 4.4 유지보수성 요구사항

#### NFR-MAIN-001: 코드 품질
- **설명**: 코드는 유지보수가 용이하도록 작성되어야 한다.
- **품질 기준**:
  - 주석 비율: 최소 20%
  - 메서드 복잡도: Cyclomatic Complexity < 10
  - 코드 중복: 최대 5%
  - 명명 규칙 준수: 100%

#### NFR-MAIN-002: 테스트 커버리지
- **설명**: 코드의 충분한 테스트 커버리지를 확보해야 한다.
- **커버리지 기준**:
  - 단위 테스트: 최소 70%
  - 통합 테스트: 주요 시나리오 100% 커버

## 6. 데이터 요구사항

### 6.1 데이터 저장

#### DR-DATA-001: 데이터 저장 관리
- **설명**: 입력 데이터와 계산 결과를 메모리에 효율적으로 저장 관리한다.
- **우선순위**: 높음
- **저장 항목**:
  - 재료 정보
  - 입력 물성값
  - 계산 결과
  - 그래프 데이터
  - 세션 정보
- **데이터 구조**: Java Collection Framework (List, Map)
- **메모리 관리**: 
  - 최대 메모리 사용량: 512MB
  - 가비지 컬렉션 최적화
- **검증 방법**: 대용량 데이터 처리 시 메모리 사용량 모니터링
- **담당 브랜치**: feature/database-setup
- **관련 요구사항**: FR-CALC-001, NFR-PERF-001

#### DR-DATA-002: 데이터베이스 스키마
- **설명**: SQLite 데이터베이스를 사용하여 영구 데이터를 저장한다.
- **우선순위**: 중간
- **테이블 구조**:
  - **materials**: 재료 정보
    - id (INTEGER, PK)
    - name (TEXT)
    - youngs_modulus (REAL)
    - yield_strength (REAL)
    - tensile_strength (REAL)
    - created_at (TIMESTAMP)
  - **experiments**: 실험 정보
    - id (INTEGER, PK)
    - material_id (INTEGER, FK)
    - specimen_diameter (REAL)
    - test_date (DATE)
    - notes (TEXT)
  - **stress_strain_data**: 응력-변형률 데이터
    - id (INTEGER, PK)
    - experiment_id (INTEGER, FK)
    - strain (REAL)
    - stress (REAL)
- **인덱스**: material_id, experiment_id에 인덱스 생성
- **검증 방법**: 데이터 무결성 제약 조건 확인
- **담당 브랜치**: feature/database-setup

### 6.2 데이터 형식

- **CSV 형식**:
  ```
  변위(mm),하중(N)

  0.2,501.0

  ```

#### 출력 데이터 형식
- **응력-변형률 데이터**: CSV, Excel
- **그래프**: PNG, JPEG
- **보고서**: PDF

### 9.1 용어 정의

|------|------|
