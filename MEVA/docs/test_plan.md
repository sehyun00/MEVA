# 테스트 계획서

## 1. 목적
본 테스트 계획서는 인장시험 프로그램의 기능 및 성능을 검증하고, 오류를 사전에 발견하여 품질을 확보하는 것을 목적으로 한다.

## 2. 테스트 범위
- 사용자 입력 검증 (형식, 범위, 논리적 일관성)
- 시험 데이터 처리 로직
- 계산 결과 검증 (이론적 한계값과 비교, 예: SS400)
- 결과 보고서 생성
- 예외 상황 처리 (파일 미존재, 잘못된 입력 등)

## 3. 테스트 접근 방식
- 단위 테스트(Unit Test) → JUnit 사용
- 통합 테스트(Integration Test) → 모듈 간 데이터 흐름 검증
- 시스템 테스트(System Test) → 실제 인장시험 시뮬레이션
- 자동화 검사 절차 설계 → 반복 테스트 가능하도록 스크립트 구성

## 4. 테스트 환경
- OS: Windows 10 / Linux
- Java: OpenJDK 17
- IDE: Eclipse / IntelliJ IDEA

## 5. 테스트 산출물
- 테스트 케이스 문서
- 테스트 결과 로그
- 버그 리포트 템플릿
